```markdown
# Java

Java ist eine plattformunabhängige, objektorientierte Programmiersprache, die 1995 von Sun Microsystems veröffentlicht wurde. Ihr Credo „Write Once, Run Anywhere“ beruht darauf, dass Java-Code vom Java Virtual Machine (JVM) interpretiert wird und somit auf jedem System mit installierter JVM lauffähig ist. Java wird in Desktop-, Web-, Mobile- und Enterprise-Anwendungen, in der Cloud sowie im Embedded-Umfeld eingesetzt.

---

## Java Intro

Java-Programme bestehen aus Klassen, in denen – anders als in C oder C++ – sämtliche Anweisungen gekapselt sind. Das erste Programm eines Anfängers ist traditionell „Hello World“. Es demonstriert die grundlegende Struktur, den Einstiegspunkt `main()` und die Konsolenausgabe.

```
// Datei: HelloWorld.java
public class HelloWorld {

    // Einstiegspunkt des Programms
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

**Erläuterung**

1. `public class HelloWorld`  
   • Definiert eine Klasse namens `HelloWorld`.  
   • Klassenname und Dateiname müssen exakt übereinstimmen.

2. `public static void main(String[] args)`  
   • Die JVM startet jedes Java-Programm in dieser Methode.  
   • `String[] args` enthält Kommandozeilenparameter.  
   • `static` ermöglicht Aufruf ohne Objekterzeugung.

3. `System.out.println(...)`  
   • Gibt eine Zeile Text auf der Konsole aus.  
   • `System.out` ist der Standard-Ausgabestrom; `println` hängt automatisch einen Zeilenumbruch an.

**Kompilieren & Ausführen**

```
javac HelloWorld.java   # erzeugt HelloWorld.class (Bytecode)
java HelloWorld         # führt das Programm in der JVM aus
```

## Java Syntax

Die Syntax von Java ähnelt C/C++, vermeidet jedoch gefährliche Features wie Zeigerarithmetik und Mehrfachvererbung von Implementierung.

| Sprachelement | Beispiel | Zweck |
|---------------|----------|-------|
| Klasse        | `class Auto {}` | Codekapselung |

| Block         | `{ ... }` | Gruppierung von Anweisungen |

Wichtige Regeln

2. Blöcke werden durch geschweifte Klammern begrenzt.  
3. Groß-/Kleinschreibung ist relevant.  
4. `package` am Dateianfang gruppiert Klassen logisch.
```

```
## Java Type Casting

**Implizites Casting (Widening)** – sicherer Übergang von kleinerem zu größerem Typ.

```
int i = 100;
long l = i;      // int → long
```

**Explizites Casting (Narrowing)** – kann Informationsverlust verursachen.

```
double d = 9.99;
int    j = (int) d;  // j = 9
```

Beim Casting zwischen Referenztypen gilt die Vererbungs- und Implementierungshierarchie; unsichere Casts sollten mit `instanceof` geprüft werden.
```

## Java Methods

Methoden kapseln wiederverwendbaren Code – sie beschreiben **Verhalten** und können Werte zurückgeben.

```
// Datei: Rechner.java
public class Rechner {

    // Eine Methode ohne Rückgabewert
    public void begruessung() {
        System.out.println("Willkommen im Rechner!");
    }

    // Eine Methode mit Rückgabewert
    public int addiere(int a, int b) {
        return a + b;
    }
}
```

**Erläuterung**

1. **Signatur**  
   -  `Sichtbarkeit Rückgabetyp Methodenname(Parameterliste)`  
   -  Der **Rückgabetyp** ist `void`, wenn nichts zurückgegeben wird.  

2. **Parameterliste**  
   -  Beinhaltet Datentyp und Bezeichner.  
   -  Mehrere Parameter werden durch Kommas getrennt.

3. **Rückgabe**  
   -  Das Schlüsselwort `return` beendet die Methode und liefert einen Wert des angegebenen Typs.  

Aufruf:

```
Rechner r = new Rechner();
r.begruessung();              // Konsolenausgabe
int summe = r.addiere(7, 5);  // summe = 12
```

## Java Method Parameters

Parameter übergeben **Eingabedaten** an Methoden.

```
// Methode mit verschiedenen Parametern
public double kreisFlaeche(double radius, final double PI) {
    // PI ist konstant und darf nicht verändert werden
    return PI * radius * radius;
}
```

Wichtige Aspekte  
-  **Positionsabhängig**: Reihenfolge der Übergabe muss stimmen.  
-  **Typensicherheit**: Compiler prüft Datentypen.  
-  **`final`-Parameter**: Wert darf innerhalb der Methode nicht verändert werden.  

Variadic Parameters (unbestimmte Anzahl):

```
public int sum(int... werte) {
    int erg = 0;
    for (int v : werte) erg += v;
    return erg;
}

sum(1, 2, 3, 4); // 10
```

## Java Method Overloading

Überladen bedeutet, dass mehrere Methoden **gleichen Namens** existieren, sich aber in **Parameterzahl oder -typen** unterscheiden.

```
public class Drucker {

    // 1) Eine Zahl drucken
    public void drucke(int zahl) {
        System.out.println(zahl);
    }

    // 2) Zwei Zahlen drucken
    public void drucke(int a, int b) {
        System.out.println(a + ", " + b);
    }

    // 3) Einen String drucken
    public void drucke(String text) {
        System.out.println(text);
    }
}
```

Auflösung erfolgt **zur Compile-Zeit** anhand des besten passenden Parametersatzes. Rückgabetyp allein reicht **nicht** zum Überladen.

## Java Scope

Der **Gültigkeitsbereich** eines Bezeichners bestimmt, wo er sichtbar ist.

| Ebene             | Deklaration                       | Lebensdauer            |
|-------------------|-----------------------------------|------------------------|
| Lokaler Scope     | Innerhalb von Methoden/Blöcken    | Bis Blockende          |
| Parameter         | In Methodenkopf                   | Während Methodenaufruf |
| Instanzvariable   | Außerhalb von Methoden, ohne `static` | Solange Objekt lebt    |
| Klassenvariable   | Mit `static` deklariert           | Während JVM-Laufzeit   |

Beispiel:

```
public class Demo {

    static int klassenZahl = 10; // Klassenvariable
    int instanzZahl = 5;         // Instanzvariable

    public void methode() {
        int lokal = 2;           // Lokale Variable
        for (int i = 0; i < lokal; i++) {
            int schleifenVar = i; // Nur innerhalb der Schleife sichtbar
            System.out.println(schleifenVar);
        }
    }
}
```

## Java Recursion

Eine Methode ruft sich **selbst** auf, bis eine **Abbruchbedingung** erfüllt ist.

```
// Fakultät berechnen: n! = n * (n-1)!
public class Fakultaet {

    public long fakultaet(long n) {
        if (n <= 1) return 1;           // Abbruchfall
        return n * fakultaet(n - 1);    // Rekursiver Aufruf
    }
}
```

Stack-Tiefe  
-  Jeder Aufruf belegt Speicher auf dem **Call-Stack**.  
-  Tiefe Rekursion kann zu `StackOverflowError` führen.  

Tail-Recursion (optimierbar):

```
public long fakultaetTR(long n, long acc) {
    if (n <= 1) return acc;
    return fakultaetTR(n - 1, n * acc);
}

// Aufruf: fakultaetTR(5, 1) → 120
```
## Java Classes

Klassen sind die **Bausteine** objektorientierter Programmierung. Sie definieren Datenfelder (**Attribute**) und Verhalten (**Methoden**) für Objekte.

```
// Datei: Auto.java
public class Auto {
    
    // Attribute (Eigenschaften)
    private String marke;
    private int baujahr;
    private double geschwindigkeit;
    
    // Konstruktor
    public Auto(String marke, int baujahr) {
        this.marke = marke;
        this.baujahr = baujahr;
        this.geschwindigkeit = 0.0;
    }
    
    // Methoden (Verhalten)
    public void beschleunigen(double delta) {
        geschwindigkeit += delta;
    }
    
    public void bremsen() {
        geschwindigkeit = 0.0;
    }
    
    public String getInfo() {
        return marke + " (" + baujahr + ") - " + geschwindigkeit + " km/h";
    }
}
```

**Erläuterung**

1. **Attribute** speichern den **Zustand** eines Objekts  
2. **Konstruktor** initialisiert neue Objekte  
3. **Methoden** definieren **Verhalten** und können Zustand verändern  
4. **private** schützt interne Daten vor direktem Zugriff  

## Java OOP

**Objektorientierte Programmierung** basiert auf vier Grundprinzipien:

| Prinzip | Beschreibung | Beispiel |
|---------|-------------|----------|
| **Kapselung** | Daten und Methoden werden in Klassen gebündelt | `Auto`-Klasse mit privaten Attributen |
| **Vererbung** | Neue Klassen erben von bestehenden | `Sportwagen extends Auto` |
| **Polymorphie** | Gleiches Interface, verschiedene Implementierungen | `fahrzeug.starten()` |
| **Abstraktion** | Komplexität wird durch Interfaces vereinfacht | `Fahrzeug`-Interface |

Vorteile:
- **Wiederverwendbarkeit** von Code
- **Modularität** und bessere Wartbarkeit
- **Erweiterbarkeit** durch Vererbung

## Java Modifiers

Modifikatoren steuern die **Sichtbarkeit** und das **Verhalten** von Klassen, Methoden und Variablen.

```php
public class ModifierDemo {
```

## Java Encapsulation

**Kapselung** versteckt interne Details und bietet kontrollierten Zugriff über Getter/Setter.

```php
public class BankKonto {
    
    // Private Attribute - von außen nicht zugänglich
    private String kontoNummer;
    private double saldo;
    private String pin;
    
    public BankKonto(String kontoNummer, String pin) {
        this.kontoNummer = kontoNummer;
        this.pin = pin;
        this.saldo = 0.0;
    }
    
    // Getter mit Validierung
    public double getSaldo(String eingabePin) {
        if (pinKorrekt(eingabePin)) {
            return saldo;
        }
        throw new IllegalArgumentException("Falsche PIN!");
    }
    
    // Setter mit Geschäftslogik
    public boolean einzahlen(double betrag, String eingabePin) {
        if (!pinKorrekt(eingabePin)) return false;
        if (betrag  saldo) return false;
        
        saldo -= betrag;
        return true;
    }
    
    // Private Hilfsmethode
    private boolean pinKorrekt(String eingabePin) {
        return this.pin.equals(eingabePin);
    }
    
    // Nur lesender Zugriff auf Kontonummer
    public String getKontoNummer() {
        return kontoNummer;
    }
}
```

**Vorteile der Kapselung**
- **Datenschutz**: Attribute sind vor direktem Zugriff geschützt
- **Validierung**: Eingaben können geprüft werden
- **Änderungssicherheit**: Interne Implementierung kann geändert werden
- **Geschäftslogik**: Regeln werden in Gettern/Settern durchgesetzt

## Java Inheritance

**Vererbung** ermöglicht es, neue Klassen basierend auf bestehenden zu erstellen.

```php
// Basis-Klasse (Superklasse)
public class Fahrzeug {
    
    protected String marke;
    protected int baujahr;
    protected double geschwindigkeit;
    
    public Fahrzeug(String marke, int baujahr) {
        this.marke = marke;
        this.baujahr = baujahr;
        this.geschwindigkeit = 0.0;
    }
    
    public void starten() {
        System.out.println(marke + " wird gestartet");
    }
    
    public void beschleunigen(double wert) {
        geschwindigkeit += wert;
        System.out.println("Neue Geschwindigkeit: " + geschwindigkeit);
    }
    
    public String getInfo() {
        return marke + " (" + baujahr + ")";
    }
}

// Abgeleitete Klasse (Subklasse)
public class Auto extends Fahrzeug {
    
    private int tueren;
    private boolean automatik;

        super.starten();  // Eltern-Methode aufrufen
        System.out.println("Auto ist fahrbereit");

    public void hupen() {
        System.out.println("Huuup!");

```

**Vererbungs-Regeln**
- **Einzelvererbung**: Klasse kann nur von einer Superklasse erben
- **`extends`**: Schlüsselwort für Vererbung
- **`super`**: Zugriff auf Eltern-Klasse
- **`@Override`**: Kennzeichnet überschriebene Methoden

## Java Polymorphism

**Polymorphie** ermöglicht es, dass verschiedene Klassen die gleiche Schnittstelle implementieren.

```php
public class PolymorphieDemo {
    
    public static void main(String[] args) {
        // Polymorphe Referenzen
        Fahrzeug[] fuhrpark = {
            new Auto("BMW", 2020, 4),
            new Motorrad("Harley", 2019, 1200),
            new LKW("MAN", 2018, 40)
        };
        
        // Gleiche Methode, verschiedene Implementierungen
        for (Fahrzeug f : fuhrpark) {
            f.starten();        // Polymorphie zur Laufzeit
            f.beschleunigen(50);
            System.out.println(f.getInfo());
            System.out.println("---");
        }
        
        // Typ-Prüfung und Casting
        for (Fahrzeug f : fuhrpark) {
            if (f instanceof Auto) {
                Auto auto = (Auto) f;
                auto.hupen();
            } else if (f instanceof Motorrad) {
                Motorrad bike = (Motorrad) f;
                bike.wheelie();
            }
        }
    }
}

class Motorrad extends Fahrzeug {
    private int hubraum;
    
    public Motorrad(String marke, int baujahr, int hubraum) {
        super(marke, baujahr);
        this.hubraum = hubraum;
    }
    
    @Override
    public void starten() {
        System.out.println("Motorrad röhrt auf!");
    }
    
    public void wheelie() {
        System.out.println("Wheelie!");
    }
}
```

**Polymorphie-Arten**
- **Überladen**: Mehrere Methoden gleichen Namens
- **Überschreiben**: Subklasse redefiniert Methode
- **Interface-Polymorphie**: Verschiedene Implementierungen

## Java Inner Classes

**Innere Klassen** (Inner Classes) sind Klassen, die innerhalb einer anderen Klasse definiert werden und Zugriff auf deren private Mitglieder haben.

```php
public class AeussereKlasse {
    
    private String geheimnis = "Vertrauliche Daten";
    private static int zaehler = 0;
    
    // 1) Member Inner Class (Nicht-statisch)
    public class InnereKlasse {
        private String nachricht = "Von innen";
        
        public void zeigeDaten() {
            System.out.println("Zugriff auf: " + geheimnis);  // Direkter Zugriff!
            System.out.println("Innere Nachricht: " + nachricht);
        }
        
        public void rufAeussereMethode() {
            aeussereMethode();  // Kann äußere Methoden aufrufen
        }
    }
    
    // 2) Static Nested Class
    public static class StatischeInnereKlasse {
        public void zeigeDaten() {
            System.out.println("Statischer Zähler: " + zaehler);
            // System.out.println(geheimnis); // FEHLER! Kein Zugriff auf Instanzvariablen
        }
    }
    
    // 3) Local Inner Class (in Methode)
    public void demonstriereLokalKlasse() {
        final String lokalVariable = "Lokal verfügbar";
        
        class LokaleKlasse {
            public void zeige() {
                System.out.println("Lokal: " + lokalVariable);
                System.out.println("Äußeres Geheimnis: " + geheimnis);
            }
        }
        
        LokaleKlasse lokal = new LokaleKlasse();
        lokal.zeige();
    }
    
    // 4) Anonymous Inner Class
    public void demonstriereAnonymKlasse() {
        Runnable task = new Runnable() {
            @Override
            public void run() {
                System.out.println("Anonyme Klasse läuft: " + geheimnis);
            }
        };
        task.run();
    }
    
    private void aeussereMethode() {
        System.out.println("Äußere Methode aufgerufen");
    }
}

// Verwendung
public class InnerClassDemo {
    public static void main(String[] args) {
        AeussereKlasse aeussere = new AeussereKlasse();
        
        // Member Inner Class erstellen
        AeussereKlasse.InnereKlasse innere = aeussere.new InnereKlasse();
        innere.zeigeDaten();
        
        // Static Nested Class erstellen
        AeussereKlasse.StatischeInnereKlasse statisch = 
            new AeussereKlasse.StatischeInnereKlasse();
        statisch.zeigeDaten();
        
        // Lokale und anonyme Klassen
        aeussere.demonstriereLokalKlasse();
        aeussere.demonstriereAnonymKlasse();
    }
}
```

**Inner Class Typen**
- **Member Inner Class**: Vollzugriff auf äußere Klasse
- **Static Nested Class**: Nur Zugriff auf statische Mitglieder
- **Local Inner Class**: Innerhalb von Methoden definiert
- **Anonymous Inner Class**: Ohne Klassennamen, oft für Interfaces

## Java Abstraction

**Abstraktion** versteckt Implementierungsdetails und zeigt nur die wesentliche Funktionalität. Sie wird durch **abstrakte Klassen** und **Interfaces** erreicht.

```php
// Abstrakte Klasse - kann nicht instanziiert werden
public abstract class Form {
    
    protected String farbe;
    protected double x, y;  // Position
    
    public Form(String farbe, double x, double y) {
        this.farbe = farbe;
        this.x = x;
        this.y = y;
    }
    
    // Abstrakte Methode - muss implementiert werden
    public abstract double berechneFlaeche();
    public abstract double berechneUmfang();
    
    // Konkrete Methode - kann vererbt werden
    public void verschieben(double dx, double dy) {
        this.x += dx;
        this.y += dy;
        System.out.println("Form verschoben zu (" + x + ", " + y + ")");
    }
    
    public String getFarbe() {
        return farbe;
    }
    
    // Template Method Pattern
    public final void zeichnen() {
        System.out.println("=== Zeichne " + getClass().getSimpleName() + " ===");
        System.out.println("Farbe: " + farbe);
        System.out.println("Position: (" + x + ", " + y + ")");
        System.out.println("Fläche: " + berechneFlaeche());
        System.out.println("Umfang: " + berechneUmfang());
        nachrichtNachZeichnen();
    }
    
    // Hook-Methode (optional überschreibbar)
    protected void nachrichtNachZeichnen() {
        System.out.println("Form gezeichnet!");
    }
}

// Konkrete Implementierung
public class Kreis extends Form {
    
    private double radius;
    
    public Kreis(String farbe, double x, double y, double radius) {
        super(farbe, x, y);
        this.radius = radius;
    }
    
    @Override
    public double berechneFlaeche() {
        return Math.PI * radius * radius;
    }
    
    @Override
    public double berechneUmfang() {
        return 2 * Math.PI * radius;
    }
    
    @Override
    protected void nachrichtNachZeichnen() {
        System.out.println("Kreis mit Radius " + radius + " gezeichnet!");
    }
}

public class Rechteck extends Form {
    
    private double breite, hoehe;
    
    public Rechteck(String farbe, double x, double y, double breite, double hoehe) {
        super(farbe, x, y);
        this.breite = breite;
        this.hoehe = hoehe;
    }
    
    @Override
    public double berechneFlaeche() {
        return breite * hoehe;
    }
    
    @Override
    public double berechneUmfang() {
        return 2 * (breite + hoehe);
    }
}
```

**Abstrakte Klassen vs. Interfaces**
- **Abstrakte Klassen**: Können konkrete Methoden enthalten, Konstruktoren haben
- **Interfaces**: Nur abstrakte Methoden (bis Java 8), keine Konstruktoren

## Java Interface

**Interfaces** definieren einen **Vertrag** (Contract), den implementierende Klassen einhalten müssen.

```php
    static boolean istGeschwindigkeitErlaubt(double geschwindigkeit) {
        return geschwindigkeit >= 0 && geschwindigkeit  10) {
            gestartet = true;
```

```php
                System.out.println("JSON-Inhalt gelesen:");

                String[] zeilen = json.split("\\n");
                for (String zeile : zeilen) {
                    if (zeile.contains("\"name\":")) {
                        String name = zeile.substring(zeile.indexOf(":") + 1).trim();
                        System.out.println("Name gefunden: " + name);
                    }
                }
            }
            
        } catch (FileNotFoundException e) {

        }
    }
```

|---------|---------------|----------|-----------|
| `Scanner` | Kleine Dateien, einfache Struktur | Einfach zu verwenden | Langsamer bei großen Dateien |
| `BufferedReader` | Große Dateien, zeilenweise | Sehr effizient | Mehr Code erforderlich |
| `Files.readAllLines()` | Moderne Anwendungen | Kompakt, UTF-8 Standard | Lädt gesamte Datei in Speicher |
| `Files.readString()` | Textdateien komplett | Ein Aufruf für gesamten Inhalt | Nur für kleinere Dateien |

**Performance-Tipps**

- **Files.readAllLines()** für kleine bis mittlere Dateien [3][4]
- **Scanner** für strukturierte Eingabe mit Tokenizing [5][6]
- **Try-with-resources** für automatisches Schließen [1][7]

## Java LinkedList

`LinkedList` implementiert sowohl das `List`- als auch das `Deque`-Interface und basiert auf einer **doppelt verketteten Liste**.

```php
import java.util.LinkedList;

public class LinkedListDemo {

    public static void main(String[] args) {
        LinkedList queue = new LinkedList<>();

        // Elemente hinten anfügen (Queue-Betrieb)
        queue.offer("Erstes");
        queue.offer("Zweites");
        queue.offer("Drittes");
        System.out.println("Queue: " + queue);

        // Elemente entnehmen
        System.out.println("poll():  " + queue.poll());   // Entfernt Kopf
        System.out.println("peek():  " + queue.peek());   // Nur ansehen
        System.out.println("Queue:   " + queue);

        // Stack-Betrieb
        queue.push("Neues Kopf-Element");
        System.out.println("push():  " + queue);
        System.out.println("pop():   " + queue.pop());
        System.out.println("Queue:   " + queue);
    }
}
```

**Wann LinkedList wählen?**

| Anforderung                 | Passend? |
|-----------------------------|----------|
| Häufiges Einfügen/Löschen **am Anfang/Ende** | ✓ |
| **Zufälliger** Indexzugriff | ✗ (langsam) |
| Speicher-Overhead gering    | ✗ (mehr Pointer) |

## Java Iterator

## Java Annotations

**Annotations** liefern Metadaten, die vom Compiler, von Frameworks oder zur Laufzeit (Reflection) ausgewertet werden können.

```php
// Eigene Annotation definieren
import java.lang.annotation.*;

@Retention(RetentionPolicy.RUNTIME)   // Lebensdauer

public @interface Testfall {
    String autor();
    int    priorität() default 1;
}

// Annotation verwenden
public class BeispielTests {

    @Testfall(autor = "Max", priorität = 2)
    public void testAddiere() {
        // …
    }

    @Deprecated                    // Vorhandene JDK-Annotation
    public void altMethode() { }
}
```

**Standard-Annotationen**

| Annotation    | Zweck                                               |
|---------------|-----------------------------------------------------|
| `@Override`   | Erzwingt Überschreiben einer Superklassen-Methode   |
| `@Deprecated` | Kennzeichnet veraltete API-Elemente                |
| `@SuppressWarnings` | Unterdrückt Compilerwarnungen                |
| `@FunctionalInterface` | Prüft, dass Interface genau **eine** abstrakte Methode besitzt |

```php
class Rechteck {

    private double breite;

        this.breite = breite;

    public double getBreite() { return breite; }

    public void setLaenge(double laenge) { this.laenge = laenge; }
    public void setBreite(double breite) { this.breite = breite; }
    
    // Berechnungen
    public double getFlaeche() {
        return laenge * breite;
    }
    
    public double getUmfang() {
        return 2 * (laenge + breite);

    public double getDiagonale() {
        return Math.sqrt(laenge * laenge + breite * breite);
    }
    
    public boolean istQuadrat() {
        return laenge == breite;
    }
    
    @Override
    public String toString() {
        return String.format("Rechteck[%.1f x %.1f] - Fläche: %.2f",
```

```php
    public static void quickSort(int[] array, int low, int high) {
        if (low < high) {
            int pi = partition(array, low, high);
            quickSort(array, low, pi - 1);
            quickSort(array, pi + 1, high);
        }
    }
    
    public static int partition(int[] array, int low, int high) {
        int pivot = array[high];
        int i = (low - 1);
        
        for (int j = low; j < high; j++) {
            if (array[j] <= pivot) {
                i++;
                int temp = array[i];
                array[i] = array[j];
                array[j] = temp;
            }
        }
        
        int temp = array[i + 1];
        array[i + 1] = array[high];
        array[high] = temp;
        
        return i + 1;
    }
    
    public static void mergeSort(int[] array, int left, int right) {
        if (left < right) {
            int middle = left + (right - left) / 2;
            mergeSort(array, left, middle);
            mergeSort(array, middle + 1, right);
            merge(array, left, middle, right);
        }
    }
    
    public static void merge(int[] array, int left, int middle, int right) {
        int n1 = middle - left + 1;
        int n2 = right - middle;
        
        int[] leftArray = new int[n1];
        int[] rightArray = new int[n2];
        
        System.arraycopy(array, left, leftArray, 0, n1);
        System.arraycopy(array, middle + 1, rightArray, 0, n2);
        
        int i = 0, j = 0, k = left;
        
        while (i < n1 && j < n2) {
            if (leftArray[i] <= rightArray[j]) {
                array[k] = leftArray[i];
                i++;
            } else {
                array[k] = rightArray[j];
                j++;
            }
            k++;
        }
        
        while (i < n1) {
            array[k] = leftArray[i];
            i++;
            k++;
        }
        
        while (j < n2) {
            array[k] = rightArray[j];
            j++;
            k++;
        }
    }
```

## Java Arrays Methods

Die Klasse **`java.util.Arrays`** bietet zahlreiche statische Hilfsmethoden, um native Arrays komfortabel zu verarbeiten.

```php
import java.util.Arrays;

public class ArraysMethodsDemo {

    public static void main(String[] args) {

        int[] daten = {5, 2, 8, 1, 9, 3};

        // sort() – aufsteigend sortieren
        Arrays.sort(daten);
        System.out.println("sort():     " + Arrays.toString(daten));

        // binarySearch() – binäre Suche (nur in sortiertem Array)
        int index = Arrays.binarySearch(daten, 8);
        System.out.println("binarySearch(8): Index = " + index);

        // copyOf() – Array verlängern oder kürzen
        int[] kopie = Arrays.copyOf(daten, 10);   // mit Nullen aufgefüllt
        System.out.println("copyOf(10): " + Arrays.toString(kopie));

        // copyOfRange() – Teilarray erstellen
        int[] teil = Arrays.copyOfRange(daten, 1, 4); // Index 1–3
        System.out.println("copyOfRange(1,4): " + Arrays.toString(teil));

        // fill() – komplettes Array mit Wert belegen
        Arrays.fill(kopie, -1);
        System.out.println("fill(-1):  " + Arrays.toString(kopie));

        // setAll() – Werte durch Lambda generieren (Java 8+)
        Arrays.setAll(kopie, i -> i * i);
        System.out.println("setAll(i²): " + Arrays.toString(kopie));

        // equals() – inhaltsgleicher Vergleich
        int[] a = {1, 2, 3};
        int[] b = {1, 2, 3};
        System.out.println("equals():  " + Arrays.equals(a, b));

        // deepEquals(), deepToString() für mehrdimensionale Arrays
        int[][] m1 = {{1, 2}, {3, 4}};
        int[][] m2 = {{1, 2}, {3, 4}};
        System.out.println("deepEquals(): " + Arrays.deepEquals(m1, m2));
        System.out.println("deepToString(): " + Arrays.deepToString(m1));
    }
}
```

**Wichtige Methodenübersicht**

| Methode | Zweck | Besonderheiten |
|---------|-------|---------------|
| `sort()` | Sortiert Array aufsteigend | Dual-Pivot-Quicksort (primitive), TimSort (Objekte) |
| `parallelSort()` | Parallele Sortierung | Nutzt Fork/Join-Pool |
| `binarySearch()` | Binäre Suche | Array muss sortiert sein |
| `copyOf()`, `copyOfRange()` | Kopien/Teilausschnitte | Größe anpassbar |
| `fill()` | Array komplett belegen | nützlich für Initialisierung |
| `setAll()` | Werte via Lambda generieren | Java 8+ |
| `equals()`, `deepEquals()` | Vergleich | `deep*` für verschachtelte Arrays |
| `toString()`, `deepToString()` | String-Darstellung | Debugging |

```php
    public static long fibIter(int n) {
        if (n >= 1; s++; }

        long x = modPow(a, d, n);
        if (x == 1 || x == n - 1) return true;
        for (int r = 1; r  0) {
            if ((exp & 1) == 1) res = modMul(res, base, mod);
            base = modMul(base, base, mod);
            exp >>= 1;
```

### 4. Mehr-Thread-Downloader (Executor & Futures)

```php
// Datei: MultiDownloader.java
import java.net.URL;
import java.nio.file.*;
import java.util.List;
import java.util.concurrent.*;

public class MultiDownloader {

    private static final List LINKS = List.of(
        "https://example.com/file1.jpg",
        "https://example.com/file2.jpg",
        "https://example.com/file3.jpg"
    );

    public static void main(String[] args) throws Exception {
        ExecutorService pool = Executors.newFixedThreadPool(4);
        List> futures =
            LINKS.stream().map(link -> pool.submit(() -> download(link))).toList();

        for (Future f : futures) {
            System.out.println("Gespeichert: " + f.get());
        }
        pool.shutdown();
    }

    private static Path download(String link) throws Exception {
        URL url = new URL(link);
        Path target = Paths.get("downloads", Path.of(url.getPath()).getFileName().toString());
        Files.createDirectories(target.getParent());
        try (var in = url.openStream()) {
            Files.copy(in, target, StandardCopyOption.REPLACE_EXISTING);
        }
        return target;
    }
}
```

**Erklärung**

* **`ExecutorService`** verwaltet Threads; **`Future`** liefert Ergebnisse asynchron.  
* Bei sehr vielen Downloads ist **`CompletableFuture`** oder **`virtual threads`** (JDK 21) eine Option.

```php
/**
 * Methode 1: String-Funktionen (strrev)
 *
 * @param string $s Eingabestring
 * @return string Umgekehrter String
 */
function reverseWithStrrev(string $s): string {
    return strrev($s);
}

/**
 * Methode 2: Manuelles Loop (UTF-8-fähig)
 *
 * @param string $s Eingabestring
 * @return string Umgekehrter String
 */
function reverseWithLoop(string $s): string {
    $chars = preg_split('//u', $s, -1, PREG_SPLIT_NO_EMPTY);
    $rev   = array_reverse($chars);
    return implode('', $rev);
}

/**
 * Methode 3: Rekursion
 *
 * @param string $s Eingabestring
 * @return string Umgekehrter String
 */
function reverseRecursive(string $s): string {
    if (mb_strlen($s) <= 1) {
        return $s;
    }
    $first = mb_substr($s, 0, 1);
    $rest  = mb_substr($s, 1);
    return reverseRecursive($rest) . $first;
}
```

```php
/**
 * Berechnet die Summe aller Werte im Array.
 *
 * @param array $arr Numerisches Array
 * @return float|int Summe der Elemente
 */
function sumArray(array $arr) {
    $total = 0;
    foreach ($arr as $value) {
        $total += $value;
    }
    return $total;
}

/**
 * Berechnet den Durchschnitt (Mittelwert) aller Werte im Array.
 *
 * @param array $arr Numerisches Array
 * @return float Durchschnitt der Elemente, 0 bei leerem Array
 */
function averageArray(array $arr) {
    $count = count($arr);
    if ($count === 0) {
        return 0;
    }
    return sumArray($arr) / $count;
}

// Beispielaufrufe
$values = [10, 20, 30, 40, 50];

echo "Array: [" . implode(", ", $values) . "]\n";

$sum = sumArray($values);
echo "Summe: $sum\n";

$avg = averageArray($values);
echo "Durchschnitt: $avg\n";
```
